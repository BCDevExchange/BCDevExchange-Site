By Loren Mullane, Community Engagement, BCDevExchange

As any coder will tell you, sometimes code breaks. Normally, fixing broken code falls on the original coder who needs to carve out time in their busy schedule. A fix can take time depending on how much other work a coder has.

But there is another way. Enter GitHub, the popular online code sharing community. Operating on the principle that many hands make light work, GitHub harnesses the power of the crowd by matching coding work with an army of coders so broken code can be fixed faster.

Here’s a real world example of GitHub in action.

‘Git’ the Advantage

For public servant and scientist turned coder Andy Teucher, broken code meant more work.

As part of the Ministry of Environment’s Environmental Reporting BC team, Andy writes R code to analyze environmental data. R code helps make sense of the data, so Environmental Reporting BC can publically report on environmental indicators in B.C.

One of Andy’s R code packages analyzed groundwater data. When the changes were made to the server that stored the data, the code broke.

A fix would have to wait, as Andy had higher priority work.

But Environmental Reporting BC had an advantage. As a BCDev pathfinder, they had shared this R code on GitHub, which the Province of B.C. joined in the spring.

Once he became aware the code was broken, Andy posted an issue in GitHub to let everyone know what had happened.

This was no mea culpa, it was an opportunity. For those not familiar with GitHub (see the 101 below), you can tag an issue as “help wanted.” Once you post an issue, it is viewable by the millions of developers on GitHub.

In this case, a civic-minded software developer read the issue, and after a conversation with Andy on GitHub, fixed the code.

Maximizing Collaboration

This is exactly the kind of behaviour the BCDevExchange encourages. By collaborating in the open with software developers, government can rapidly develop quality tech solutions.

In this case, a generous coder was prepared to offer their help for free. For that, Andy and the BC Developer’s Exchange team are really grateful – especially for how fast the fix happened.

How can we encourage developers to collaborate on government IT projects?

What’s required for this scenario to be more commonplace? We applaud those civic-minded developers who volunteer their time and expertise to help make government better. But we recognize that developers may not want to freely contribute to every government GitHub project.

So we are exploring options to encourage developers to collaborate so development can happen more quickly than is currently possible and be rewarded for their effort.

What do you think? How could developers be rewarded for their efforts? Can there be rewards while allowing for the same kind of rapid development that occurred in this example?

GitHub 101

GitHub is a powerful online tool to collaborate with software developers.

Repos

If you want to collaborate with other developers to improve your software application, everything starts with a GitHub repository, or ‘repo’. Create a repo and share the master software code for your application under an open-source license, which grants developers permission to work on the code.

Issues

The next step is to post an issue in the repo. This gives developers an outline of the work to be done.  Examples include asking for help to fix a software bug or add a new feature to your application. Once developers know what needs to be done, they make a copy of your code, or ‘fork’ it. They modify it with new code to address the issue identified (e.g. fix the bug, add a feature, etc.).

Pull Requests

When the developer has finished the work, they notify you by sending a ‘pull’ request with the modified code. The final step is for you to review the developer’s proposed changes and to decide whether or not to accept the modified code to replace your master code.
