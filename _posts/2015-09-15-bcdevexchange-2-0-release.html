By Loren Mullane, Community Engagement, BCDevExchange

Today, we launched a new iteration of the BCDevExchange website with new features to help B.C.’s public and tech sectors collaborate and experiment together.

We believe collaboration between B.C.’s public and tech sector is a win-win – it will lead to better information technology products and services, which can be used by British Columbians, and help seed growth in one of the province’s key economic sectors.

So what are the new features? The significant change is the introduction of profiles of early adopters who embrace the BCDevExchange vision.

Meet the Early Adopters

Each early adopter has a profile page – a place where you will find digital resources and related documentation, and in some cases invitations to collaborate on technology development. Without further ado, our early adopters are:

The BC Laws API

The Queen’s Printer of British Columbia has shared their BC Laws API, an XML feed of all provincial legislation, under a license that empowers commercial use for some time. As part of the BCDevExchange they are using the WSO2 open source API manager to experiment with a new way to present their API. We think developers will be keenly interested in this approach to explore an API. Check it out here and let us know what you think.

The BC Laws API team also started a GitHub repo with an eye towards future collaborative development. This may include improving the BC Laws API, and exploring an API for municipal bylaws. A standardized B.C. platform for legislative information would allow developers to re-use functionality across jurisdictions in the province and offer value-added applications for citizens attempting to interact with the laws and bylaws that govern them. We all love PDFs but think of the value to small business owners of smartphone-enabled local bylaws, instead of trying to find and read a PDF on a smartphone.

City of Kamloops

The City of Kamloops has joined the BCDevExchange, profiling their open data which is now under the Open Government License – British Columbia. They are experimenting with us on presenting data via API and looking for opportunities to collaborate with the local tech community. Collaboration is already underway. Driven by the tech accelerator Kamloops Innovation, developers in Kamloops are exploring the potential of developing a waste reminder pickup service for city residents. We look forward to watching this project progress.

BC Data Catalogue

The team at DataBC is profiling the BC Data Catalogue, both the catalogue content, and the Province of B.C.’s code contributions. Built on the open source software project CKAN code base, the BC Data Catalogue’s code is available under an open source license on GitHub. The future includes exploring open development to improve the catalogue and give back to the community. For more details about the launch of the catalogue read this blog and be sure to check out the profile page.

BC Physical Address Geocoder

Also from DataBC, is a geocoding web service which offers a REST API. Using data from a variety of government sources and an innovative algorithm, the Geocoder turns addresses into physical locations (i.e. longitude and latitude points) and standardizes the address text as a useful by-product.  Learn more about the Geocoder API on the profile page, and check out the experiment with WSO2 as an API manager.

City of Surrey

A leader in the provincial open data space, the City of Surrey is staying nimble. Their profile page highlights the data in their open data catalogue, which can be accessed by APIs. Like the BC Data Catalogue, their catalogue code is based on CKAN, so you can find their code contributions in GitHub. They are also exploring open development in the future.

Open511 – DriveBC

For all the news on the big release of the Ministry of Transportation and Infrastructure’s Open 511 API, read the news release. Developed in partnership with the non-profit Open North, Open 511 is an API standard for provincial highway travel information, including road closures, planned work and extreme weather conditions. It is released under a license that grants commercial use. Check out their production API and an experimental version in WSO2.

User Testing of Profile Pages

In keeping with the BCDevExchange ethos of collaboration, we have not developed the concept or design of the profile pages in a vacuum. We conducted lightweight user testing on the profile pages with developers. They are the primary audience, so it’s important that the profile pages are useful to them. We will continue to user test and refine these pages.
